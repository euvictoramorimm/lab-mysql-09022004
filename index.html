Grupo A — Stored ProcedureTestado

A procedure sp_relatorio_2025 percorre os pacientes com um cursor e conta, para cada um, as consultas de 2025. Se receber um paciente_id inexistente, interrompe com SIGNAL SQLSTATE '45000'.

sp_relatorio_2025 · cursor + SIGNAL 
DELIMITER //
CREATE PROCEDURE sp_relatorio_2025(IN p_paciente_id INT)
BEGIN
  DECLARE v_done INT DEFAULT 0;
  DECLARE v_id INT; DECLARE v_nome VARCHAR(100);
  DECLARE v_qtd INT; DECLARE v_existe INT DEFAULT 0;
  DECLARE cur CURSOR FOR
     SELECT id, nome FROM pacientes
     WHERE p_paciente_id IS NULL OR id = p_paciente_id ORDER BY id;
  DECLARE CONTINUE HANDLER FOR NOT FOUND SET v_done = 1;

  IF p_paciente_id IS NOT NULL THEN
     SELECT COUNT(*) INTO v_existe FROM pacientes WHERE id = p_paciente_id;
     IF v_existe = 0 THEN
        SIGNAL SQLSTATE '45000'
          SET MESSAGE_TEXT = 'Paciente inexistente no schema lab_09022004';
     END IF;
  END IF;

  CREATE TEMPORARY TABLE tmp_rel (paciente_id INT, nome VARCHAR(100), consultas_2025 INT);

  -- percorre os pacientes com o cursor e conta as consultas de 2025
  OPEN cur;
  read_loop: LOOP
    FETCH cur INTO v_id, v_nome;
    IF v_done THEN LEAVE read_loop; END IF;
    SELECT COUNT(*) INTO v_qtd FROM consultas
      WHERE paciente_id = v_id AND YEAR(data_consulta) = 2025;
    INSERT INTO tmp_rel VALUES (v_id, v_nome, v_qtd);
  END LOOP;
  CLOSE cur;

  SELECT * FROM tmp_rel ORDER BY paciente_id;
END //
DELIMITER ;

Grupo B — Índice composto & EXPLAINObrigatório

Uma consulta filtra consultas por paciente e faixa de data. Comparando o plano de execução antes e depois do índice composto, dá para ver o otimizador trocar de estratégia.

consulta analisada + índice composto 
EXPLAIN SELECT id, especialidade FROM consultas
WHERE paciente_id = 1 AND data_consulta >= '2025-01-01';

-- ordem importa: igualdade primeiro, faixa depois
CREATE INDEX idx_consultas_paciente_data
  ON consultas (paciente_id, data_consulta);

04

Reflexão

O índice composto foi usado. Confirmei pela coluna key do EXPLAIN, que passou de paciente_id — o índice implícito criado pela chave estrangeira — para idx_consultas_paciente_data. O type mudou de ref para range e as linhas estimadas caíram de 60 para 24, com o filtered subindo de 33% para 100%. O índice composto (paciente_id, data_consulta) resolve os dois filtros — paciente e faixa de data — de uma vez, em vez de varrer as datas linha a linha. A ordem das colunas acompanha o WHERE: igualdade primeiro, faixa depois.
